### 1. 데이터 증가로 인한 응답 중단

#### 📌 문제 상황 및 진단

- 블록체인 네트워크에서 데이터를 가져올 때 부하 발생
- 이벤트가 많을 시 응답 지연 및 중단
- 데이터 누락 및 비순차적 처리로 인한 데이터 정합성 문제
- 부하가 가중되는 곳 확인

### 🛠 해결 방법
1. Event Listening
    - 이벤트를 실시간으로 구독할 수 있게 로직 변경
    - 데이터 분산성과 확장성 고려하여 Queue(RabbitMQ) 이용
    - 놓친 데이터 관리를 위한 확정된 데이터 마킹로직 추가
2. Logging & Monitoring
    - 데이터 문제 지점을 쉽게 구분하고 확인하기 위한 Logging(Elastic Search)
    -데이터의 문제 없음을 확인하기 위한 주기적 정합성 모니터링 알람(telegram)
3. Caching
    - 지연 상황에서도 최소한의 데이터를 보여줄 수 있게 우회

### 2. 데이터 처리 소요시간 증가

#### 📌 문제 상황 및 진단

가져온 데이터를 필터링하고 파싱하는데 소요되는 시간이 많이 걸림
처리되는 순서가 중요한 데이터들이 있음

#### 🛠 해결 방법
1. 분산처리
- 서버를 추가/제거한 뒤 Queue에 연결시켜 유연하게 확장할 수 있게 구조 변경

2. Ordering & Buffer
- 데이터를 분산 처리 후 결과를 Buffer에 보내어 순차적으로 처리
- Buffer가 일정 이상으로 쌓이지 않게 범위를 나누어 처리

3. Monitoring
- Buffer가 쌓이거나 일정 범위에 특정시간 이상으로 머물 시 모니터링 알람
