## 코드 철학
```
읽기 쉽고, 변경하기 쉬운 코드가 좋은 코드라 생각합니다.

시간이 지나도 이해되고 수정할 수 있어야  
지속 가능한 소프트웨어가 될테니까.
```

#### 1. 테스트 코드 작성 철학
- 작고 명확하게 유지하여 지속 가능하도록
- 단순 input, output이 아닌 함수의 실제 의미가 담길 수 있도록
#### 2. 코드 리뷰에서 중요하게 보는 점
- 복잡하지 않은지
- 독립되어 있는지, 영향력의 정도
#### 3. 기술 부채를 다루는 전략
- 빠른 개발이 필요하면 부채를 허용하되, 최대한 단순화 시키고 분리시킨다. 
- 여러 부채를 동시에 허용할 시 문제가 커지기에 부채를 지속적으로 쌓는건 문제가 있다.
- 일하는 동료들이 다 같이 정리가 된 채 나누면 더 빠르게 부채를 청산할 수 있다.
